# HabiTora 🐯

Tiger-themed habit tracker that runs 100% in the browser. Cute, fast, and private.

![HabiTora](social-preview.png)

Live (GitHub Pages)
- https://yokko405.github.io/yokos-garden/

## Features
- Cute tiger mascot with animations and messages
- Habit list with: add, complete, delete
- Gamified progression: EXP, level, coins, badges, shop items
- **Zoo Mode (New!)**: Transform your Tiger into a Penguin, Elephant, or Giraffe! 🦁🐧🐘🦒
- **Long-term Streaks**: New badges for 50, 100, 200, 365 days!
- Daily progress and streak indicator
- **Cheer Plaza (Prototype)**: Share gentle progress signals, connect by friend ID, and send supportive stamps
- **Cloud Sync (Prototype)**: Supabase login, public nickname, first-login migration card, cloud save/restore, conflict guard, and sync status
- Background music toggle and completion chime (Web Audio)
- Data saved locally via localStorage, with optional Supabase cloud backup/sync
- Import/Export JSON backup

Cloud sync migration design:
- `docs/cloud-sync-design.md`
- `docs/auth-provider-setup.md`

## Usage
1. Open the live link above
2. Add a habit with name/emoji/points/frequency
3. Tap 達成！ to complete and earn EXP/coins
4. Toggle music from the header (♪)
5. Use 設定 → クラウド保存 / クラウド復元 to back up or restore through Supabase
6. Export/Import from the UI to keep an extra manual backup

## iPhone local data migration
If the iPhone already has the real local data, treat that iPhone as the source of truth until the first successful upload.

1. Deploy this app to the same live URL the iPhone already uses: `https://yokko405.github.io/yokos-garden/`
2. Open that same URL or installed PWA on the iPhone
3. Log in from 設定
4. Tap `端末データを保存`
5. If a server revision already exists, tap `端末データでサーバー上書き`
6. Confirm `同期済み / サーバー正本`

## Development
This app ships as a static site, but the React code is bundled locally for production safety.

Run locally:

```bash
npm install
npm run build
npm run dev       # serves http://localhost:3000/
# Tests use `node scripts/serve-static.mjs . 3000` for local serving.
```

OAuth and magic link auth should be tested from `http://localhost:3000/` or the published HTTPS URL. Opening `index.html` directly with `file://` is fine for UI checks, but not for login redirects. Google/Apple buttons follow the live Supabase provider settings; magic link remains available while those providers are not configured.

Structure:
- `index.html`: app shell, styles, and local bundle reference
- `src/app.jsx`: React source code
- `app.bundle.js`: production bundle generated by `npm run build`
- `scripts/serve-static.mjs`: dependency-free local static server for tests
- `.nojekyll`: disable Jekyll on GitHub Pages
- `.gitignore`: common ignores
